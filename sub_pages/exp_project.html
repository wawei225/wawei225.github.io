A/B Testing Project

Introduction

This simulation aims to demonstrate the A/B testing process. Therefore, the data is simulated, and the website is limited to two main pages for simplicity. The methodology for data simulation and the conceptual structure of the website will be explained in detail in next sections. This project simulates a typical A/B testing process, focusing on the evaluation of a redesigned trending section on an e-commerce platform’s homepage. The primary objective is to determine whether the new design improves user engagement and increases the click-through rate (CTR) of the trending section.

Problem Statement and Hypothesis

We assume there are two main pages: the homepage and the item page. Each page consists of two sections: the discovery and trending sections for the homepage, and the description and review sections for the item page. The new UI design is for the trending section on the homepage. The trending section highlights the most-ordered items on the platform. However, empirical data indicates that its conversion rate is below the overall homepage average, suggesting that the current design does not effectively capture user interest. The product team hypothesizes that a redesigned layout will enhance visual appeal and usability, leading to a higher click-through rate.

Hypothesis:The redesigned trending section will result in a statistically significant increase in CTR compared to the existing design, thereby improving user engagement and potentially driving higher conversions.

Scope of the experiment

To simplify the scope of this A/B test, the website will be structured into two main pages and their key sections:

  • Homepage
    • Discovery Section: General recommendations based on user preferences. This section displays multiple different items and users can click on any of them, which will direct them to the item page.
    • Trending Section: Items with the highest order volume on the platform, and users can click on any of them, which will direct them to the item page.
  • Item Page
    • Description Section: Product details and specifications.
    • Review Section: User-generated reviews and ratings.

Simulated data

To conduct this simulation, historical and experimental data were synthetically generated using ChatGPT. The dataset was designed to replicate real-world user interactions and engagement patterns, ensuring the validity of the simulated A/B test.

Historical Data Generation
ChatGPT was used to create a dataset that represents prior user behavior on the platform for the two pages (Home and Item). This dataset serves as a baseline for conversion rate analysis before introducing experimental changes.

Experiment Data Generation
Simulated user interactions were generated for both:

  • Control Group (existing UI design)
  • Variant Group (redesigned UI design)

    • These interactions track user engagement across devices and sections, enabling accurate measurement of the impact of the redesigned UI on conversion rates.

    Experiment Metrics

    When determining the success metric for an A/B test, a data scientist collaborates with the product manager to align measurement with business objectives. In our experiment, we aim to evaluate changes in the trending section’s click-through rate (CTR) to measure its impact on user engagement. Given that repeated exposures may lead to diminishing returns, we select User-Level CTR—the proportion of users who click at least once—as the primary metric. This ensures we capture whether the feature effectively drives initial engagement, rather than penalizing it for natural drop-offs in repeated visits.

    The User-Level CTR is calculated as:

    \[ \text{User-Level CTR} = \frac{\text{Number of users who clicked at least once}}{\text{Total number of users exposed to the trending section}} \]

    To validate randomization, we check for Sample Ratio Mismatch (SRM) and establish a baseline User-Level CTR from historical data. This metric directly ties to our hypothesis: if the new design improves engagement, we should observe a statistically significant increase in User-Level CTR without compromising traffic distribution fairness.
